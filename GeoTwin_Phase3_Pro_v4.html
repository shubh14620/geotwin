SAR Flood Detection Methodology

1. Data Acquisition

Platform: Sentinel-1 GRD (Ground Range Detected) · Polarization: VV · Resolution: 10m × 10m
Source: Google Earth Engine COPERNICUS/S1_GRD

2. Pre-processing Pipeline

Raw GRD → Apply Orbit File → Border Noise Removal → Thermal Noise Removal → Radiometric Calibration (σ°) → Speckle Filtering (Lee / Refined Lee) → Terrain Correction (Range-Doppler) → Convert to dB: σ°(dB) = 10 × log₁₀(σ°)

3. Flood Classification

Method: Threshold-based backscatter analysis
Principle: Water bodies exhibit significantly lower backscatter due to specular reflection
If σ°(VV) < threshold_dB → FLOODED Else → NON-FLOODED

4. Post-processing

Morphological opening · Minimum mapping unit: 3×3 pixel block · Permanent water body masking (JRC GSW)

References

Twele et al. (2016) · Martinis et al. (2015) · Sentinel-1 Mission Guide, ESA

NDVI Crop Health Monitoring Methodology

1. Data Acquisition

Platform: Sentinel-2 MSI Level-2A · Bands: B4 (Red, 665nm) + B8 (NIR, 842nm) · Resolution: 10m
Source: GEE COPERNICUS/S2_SR_HARMONIZED

2. Pre-processing

L2A Product → Cloud Masking (SCL / s2cloudless) → Atmospheric Correction (already L2A) → Scale Reflectance (÷ 10000) → Clip to AOI

3. NDVI Calculation

NDVI = (NIR - RED) / (NIR + RED) = (B8 - B4 ) / (B8 + B4 ) Range: -1.0 (water/bare) → +1.0 (dense vegetation)

4. Classification

NDVI < 0.2 → Low Vegetation 0.2 ≤ NDVI < 0.5 → Moderate Vegetation NDVI ≥ 0.5 → Healthy Vegetation

5. Temporal Monitoring

Multi-temporal composites (monthly median) · Change detection · Crop phenology tracking

References

Rouse et al. (1974) · Tucker (1979) · Drusch et al. (2012) · ESA Sentinel-2 Technical Guide

LULC Classification Methodology

1. Input Data

Sentinel-2 L2A — Bands: B2,B3,B4,B8,B11,B12 · Cloud Mask applied · 10m resolution

2. Feature Stack

features = [B2, B3, B4, B8, B11, B12, NDVI, NDBI, MNDWI, EVI, BSI, NDRE] → 12 input channels per pixel

3. Classifiers (GEE)

Method 1: Random Forest (ee.Classifier.smileRandomForest) → 100 trees, max depth 15 Method 2: SVM (ee.Classifier.libsvm) → RBF kernel, C=10, gamma=0.01 Method 3: CART Decision Tree → For comparison & interpretability

4. Classes

Urban · Agriculture · Forest/Dense Veg · Shrubland · Waterbody · Barren · Fallow Land

5. Accuracy

Overall Accuracy: ~88% · Kappa: 0.84 · Validated via Google Earth & field samples

Land Surface Temperature Retrieval — Landsat-8/9

Satellite Source

Landsat-8 OLI/TIRS · Band 10 (Thermal Infrared) · 100m resampled to 30m

Algorithm: Mono-Window

Step 1: Top-of-Atmosphere Radiance L = ML × Qcal + AL Step 2: Brightness Temperature (BT) BT = K2 / ln(K1/L + 1) − 273.15 Step 3: NDVI-based Emissivity Pv = ((NDVI − NDVImin)/(NDVImax − NDVImin))² ε = 0.004×Pv + 0.986 Step 4: Land Surface Temperature LST = BT / (1 + (λ×BT/ρ) × ln(ε))

Constants

λ = 10.8 μm (Band 10) · ρ = 1.438×10⁻² m·K · K1=774.89, K2=1321.08

Change Detection Techniques

1. Image Differencing

ΔIndex = Index_T2 − Index_T1 Change if |ΔIndex| > threshold (e.g., 0.15 for NDVI)

2. Post-Classification Comparison

Classify T1 & T2 separately → compare class maps → generate change matrix

3. CVA (Change Vector Analysis)

Magnitude = √(ΔB1² + ΔB2² + ... + ΔBn²) Direction = atan2(ΔB2, ΔB1)

4. Normalized Burn Ratio (for fire)

NBR = (NIR − SWIR2) / (NIR + SWIR2) dNBR = NBR_pre − NBR_post → burn severity

Applications

Urban expansion · Deforestation · Flood damage · Post-disaster recovery · Agricultural monitoring
