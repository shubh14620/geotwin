GEOTWIN PRO · Field Introduction
Remote Sensing, GIS & Earth Observation
The three pillars that power every layer of GEOTWIN PRO — from satellite imagery acquisition to spatial analysis and intelligent dashboards.
🛰️
Remote Sensing
Layer 1
Remote Sensing is the science of acquiring information about the Earth's surface without direct physical contact — using sensors mounted on satellites, aircraft, or drones. These sensors detect electromagnetic radiation reflected or emitted by the ground, producing imagery across multiple spectral bands (Visible, NIR, SWIR, Thermal, Microwave).

In GEOTWIN PRO, RS data from Sentinel-1 (SAR), Sentinel-2 (MSI), and Landsat-8/9 is processed to compute indices like NDVI, NDWI, NDBI, and LST — revealing vegetation health, water bodies, built-up areas, and urban heat islands with centimetre-to-metre precision across any region on Earth.
Sentinel-1 SAR Sentinel-2 MSI Landsat-8/9 Multispectral Bands Spectral Indices SAR Backscatter Passive & Active Sensors Electromagnetic Spectrum
🗺️
Geographic Information System (GIS)
Layer 2
GIS is a framework for capturing, storing, manipulating, analysing, and presenting all types of geographical data. It links location-based data (coordinates, boundaries, elevation) with attribute data (land use, population, temperature) to create interactive spatial intelligence.

GEOTWIN PRO integrates GIS principles through Shapefile QA (client-side geometry validation), Leaflet.js interactive maps, QGIS-compatible export workflows, and multi-ROI (Region of Interest) analysis — letting you overlay, query, and inspect any Area of Interest with sub-metre coordinate accuracy.
Shapefile / GeoJSON Vector & Raster CRS / Projection Leaflet.js Maps QGIS Integration Spatial Query AOI Selection PostGIS
🌍
Earth Observation (EO)
Layer 3
Earth Observation is the broadest discipline — encompassing the systematic, continuous monitoring of the planet using all remote sensing technologies and data streams to inform decision-making in agriculture, disaster management, urban planning, climate science, and environmental policy.

GEOTWIN PRO brings EO to your browser: LULC Classification (7-class Random Forest), Flood Inundation Mapping (SAR Sentinel-1), Urban Heat Islands (Landsat-9 TIRS), Change Detection (NDVI differencing 2015–2024), and Google Earth Engine-backed timelapse — making satellite-grade intelligence accessible without installing a single desktop application.

Sentinel-1 SAR Flood Inundation Detection — Methodology

1. Data Acquisition

Platform: Sentinel-1 GRD (Ground Range Detected) · Polarization: VV · Resolution: 10m × 10m
Source: Google Earth Engine COPERNICUS/S1_GRD

2. Pre-processing Pipeline

Raw GRD → Apply Orbit File → Border Noise Removal → Thermal Noise Removal → Radiometric Calibration (σ°) → Speckle Filtering (Lee / Refined Lee) → Terrain Correction (Range-Doppler) → Convert to dB: σ°(dB) = 10 × log₁₀(σ°)

3. Flood Classification

Method: Threshold-based backscatter analysis
Principle: Water bodies exhibit significantly lower backscatter due to specular reflection
If σ°(VV) < threshold_dB → FLOODED Else → NON-FLOODED

4. Post-processing

Morphological opening · Minimum mapping unit: 3×3 pixel block · Permanent water body masking (JRC GSW)

References

Twele et al. (2016) · Martinis et al. (2015) · Sentinel-1 Mission Guide, ESA

NDVI Crop Health Monitoring Methodology

1. Data Acquisition

Platform: Sentinel-2 MSI Level-2A · Bands: B4 (Red, 665nm) + B8 (NIR, 842nm) · Resolution: 10m
Source: GEE COPERNICUS/S2_SR_HARMONIZED

2. Pre-processing

L2A Product → Cloud Masking (SCL / s2cloudless) → Atmospheric Correction (already L2A) → Scale Reflectance (÷ 10000) → Clip to AOI

3. NDVI Calculation

NDVI = (NIR - RED) / (NIR + RED) = (B8 - B4 ) / (B8 + B4 ) Range: -1.0 (water/bare) → +1.0 (dense vegetation)

4. Classification

NDVI < 0.2 → Low Vegetation 0.2 ≤ NDVI < 0.5 → Moderate Vegetation NDVI ≥ 0.5 → Healthy Vegetation

5. Temporal Monitoring

Multi-temporal composites (monthly median) · Change detection · Crop phenology tracking

References

Rouse et al. (1974) · Tucker (1979) · Drusch et al. (2012) · ESA Sentinel-2 Technical Guide

EVI vs NDVI Comparison — Dense Canopy Regions

✅ EVI Advantages

  • Reduces atmospheric aerosol effects via blue band
  • Reduces soil background noise (canopy correction)
  • No saturation in dense tropical forests (NDVI saturates at ~0.8)
  • Better sensitivity for LAI > 3 m²/m²
  • MODIS MOD13A1 standard product — 500m/16-day

⚠️ When to Use NDVI Instead

  • Sparse arid/semi-arid vegetation (EVI overestimates)
  • No blue band available (Sentinel-1 / Landsat-5)
  • Long time series pre-2000 (MODIS not available)
  • Simple crop monitoring (NDVI sufficient)

EVI GEE Script — Sentinel-2 + MODIS
// ═══ EVI — Enhanced Vegetation Index · GEE ═══ // Sentinel-2 EVI var s2 = ee.ImageCollection('COPERNICUS/S2_SR_HARMONIZED') .filterBounds(geometry).filterDate('2024-01-01','2024-12-31') .filter(ee.Filter.lt('CLOUDY_PIXEL_PERCENTAGE', 10)) .median().divide(10000); var EVI_S2 = s2.expression( '2.5 * ((NIR - RED) / (NIR + 6 * RED - 7.5 * BLUE + 1))', { 'NIR': s2.select('B8'), 'RED': s2.select('B4'), 'BLUE': s2.select('B2') }).rename('EVI'); Map.addLayer(EVI_S2, {min:0, max:0.8, palette:['#7f1d1d','#fbbf24','#86efac','#16a34a','#052e16']}, 'EVI S2'); // MODIS EVI (500m, 16-day composite) var modisEVI = ee.ImageCollection('MODIS/061/MOD13A1') .filterDate('2024-01-01','2024-12-31') .filterBounds(geometry) .select('EVI') .median() .multiply(0.0001); // Scale factor Map.addLayer(modisEVI, {min:0, max:0.8, palette:['#7f1d1d','#fbbf24','#86efac','#16a34a','#052e16']}, 'MODIS EVI'); // Dense canopy mask (EVI > 0.5) var denseCanopy = EVI_S2.gt(0.5).selfMask(); Map.addLayer(denseCanopy, {palette:['#052e16']}, 'Dense Canopy');

Land Surface Temperature Retrieval — Landsat-8/9

Satellite Source

Landsat-8 OLI/TIRS · Band 10 (Thermal Infrared) · 100m resampled to 30m

Algorithm: Mono-Window

Step 1: Top-of-Atmosphere Radiance L = ML × Qcal + AL Step 2: Brightness Temperature (BT) BT = K2 / ln(K1/L + 1) − 273.15 Step 3: NDVI-based Emissivity Pv = ((NDVI − NDVImin)/(NDVImax − NDVImin))² ε = 0.004×Pv + 0.986 Step 4: Land Surface Temperature LST = BT / (1 + (λ×BT/ρ) × ln(ε))

Constants

λ = 10.8 μm (Band 10) · ρ = 1.438×10⁻² m·K · K1=774.89, K2=1321.08

Change Detection Techniques

1. Image Differencing

ΔIndex = Index_T2 − Index_T1 Change if |ΔIndex| > threshold (e.g., 0.15 for NDVI)

2. Post-Classification Comparison

Classify T1 & T2 separately → compare class maps → generate change matrix

3. CVA (Change Vector Analysis)

Magnitude = √(ΔB1² + ΔB2² + ... + ΔBn²) Direction = atan2(ΔB2, ΔB1)

4. Normalized Burn Ratio (for fire)

NBR = (NIR − SWIR2) / (NIR + SWIR2) dNBR = NBR_pre − NBR_post → burn severity

🛰️ Reproducible GEE Workflows: All scripts below are production-grade. Copy → Paste in code.earthengine.google.com → Update the geometry variable with your AOI coordinates → Run. Cloud masking, atmospheric correction, and export pipelines are pre-configured.
NDVI Time Series — Sentinel-2
// ═══ NDVI MONITORING — Sentinel-2 ═══ // Paste in GEE Code Editor & change geometry var geometry = ee.Geometry.Rectangle([83.2, 26.5, 83.9, 27.1]); // Change to your AOI var startDate = '2024-01-01'; var endDate = '2024-12-31'; // Load Sentinel-2 SR with cloud mask var s2 = ee.ImageCollection('COPERNICUS/S2_SR_HARMONIZED') .filterBounds(geometry) .filterDate(startDate, endDate) .filter(ee.Filter.lt('CLOUDY_PIXEL_PERCENTAGE', 20)) .map(function(img) { var cloud = img.select('SCL').neq(3).and(img.select('SCL').neq(8)).and(img.select('SCL').neq(9)); return img.updateMask(cloud).divide(10000); }); // Compute NDVI var addNDVI = function(img) { var ndvi = img.normalizedDifference(['B8','B4']).rename('NDVI'); return img.addBands(ndvi); }; var s2_ndvi = s2.map(addNDVI); // Monthly median composite var months = ee.List.sequence(1, 12); var monthly = ee.ImageCollection.fromImages(months.map(function(m) { return s2_ndvi.filter(ee.Filter.calendarRange(m, m, 'month')) .median().set('month', m); })); // Chart var chart = ui.Chart.imageCollection.byRegion( monthly.select('NDVI'), geometry, ee.Reducer.mean(), 500, 'month') .setChartType('LineChart') .setOptions({title: 'Monthly Mean NDVI', vAxis: {title: 'NDVI'}, hAxis: {title: 'Month'}}); print(chart); // Visualize latest composite var latest = s2_ndvi.median(); Map.centerObject(geometry, 11); Map.addLayer(latest.select('NDVI'), {min:0, max:0.8, palette:['red','yellow','green']}, 'NDVI'); Map.addLayer(geometry, {color:'cyan'}, 'AOI', false, 0.5);
Flood Detection — Sentinel-1 SAR
// ═══ FLOOD DETECTION — Sentinel-1 SAR ═══ var geometry = ee.Geometry.Rectangle([83.2, 26.5, 83.9, 27.1]); // Change AOI var preDate = ['2024-06-01', '2024-06-30']; // Pre-flood period var postDate = ['2024-07-01', '2024-07-31']; // Post-flood / flood period // Load Sentinel-1 GRD function getSAR(startDate, endDate) { return ee.ImageCollection('COPERNICUS/S1_GRD') .filterBounds(geometry) .filterDate(startDate, endDate) .filter(ee.Filter.eq('instrumentMode', 'IW')) .filter(ee.Filter.listContains('transmitterReceiverPolarisation', 'VV')) .select('VV') .mean(); } var pre = getSAR(preDate[0], preDate[1]); var post = getSAR(postDate[0], postDate[1]); // Threshold-based flood detection var THRESHOLD = -16; // dB — adjust based on region var flood = post.lt(THRESHOLD).and(pre.gt(THRESHOLD)); // Remove permanent water (JRC) var jrc = ee.Image('JRC/GSW1_4/GlobalSurfaceWater').select('seasonality'); var permanentWater = jrc.gte(10); var floodMasked = flood.and(permanentWater.not()); // Stats var floodArea = floodMasked.multiply(ee.Image.pixelArea()).reduceRegion({ reducer: ee.Reducer.sum(), geometry: geometry, scale: 10, maxPixels: 1e10 }); print('Flood area (m²):', floodArea); // Visualize Map.centerObject(geometry, 11); Map.addLayer(post, {min:-25, max:0}, 'SAR Post (VV)'); Map.addLayer(floodMasked, {min:0, max:1, palette:['white','blue']}, 'Flood Extent'); Map.addLayer(pre, {min:-25, max:0}, 'SAR Pre (VV)', false); // Export Export.image.toDrive({ image: floodMasked.byte(), description: 'Flood_Detection', folder: 'GEE_Exports', region: geometry, scale: 10, crs: 'EPSG:4326' });
LULC Classification — Random Forest
// ═══ LULC — Random Forest Classification ═══ var geometry = ee.Geometry.Rectangle([83.2, 26.5, 83.9, 27.1]); var year = '2024'; // Sentinel-2 composite with indices var s2 = ee.ImageCollection('COPERNICUS/S2_SR_HARMONIZED') .filterBounds(geometry).filterDate(year+'-01-01', year+'-12-31') .filter(ee.Filter.lt('CLOUDY_PIXEL_PERCENTAGE', 15)) .median().divide(10000).clip(geometry); var addIndices = function(img) { var ndvi = img.normalizedDifference(['B8','B4']).rename('NDVI'); var ndbi = img.normalizedDifference(['B11','B8']).rename('NDBI'); var mndwi = img.normalizedDifference(['B3','B11']).rename('MNDWI'); var evi = img.expression('2.5*(NIR-RED)/(NIR+6*RED-7.5*BLUE+1)', {NIR:img.select('B8'),RED:img.select('B4'),BLUE:img.select('B2')}).rename('EVI'); return img.addBands([ndvi, ndbi, mndwi, evi]); }; var composite = addIndices(s2); // Training bands var bands = ['B2','B3','B4','B8','B11','B12','NDVI','NDBI','MNDWI','EVI']; // === DRAW TRAINING SAMPLES IN GEE MAP PANEL === // Use geometry tools (top-left) to draw polygons for each class: // Urban(0), Agriculture(1), Forest(2), Water(3), Bare(4) // Then replace these with your drawn geometries: var urban = ee.FeatureCollection(/* draw urban polygons */).map(function(f){return f.set('class',0)}); var agri = ee.FeatureCollection(/* draw agri polygons */).map(function(f){return f.set('class',1)}); var forest = ee.FeatureCollection(/* draw forest polygons */).map(function(f){return f.set('class',2)}); var water = ee.FeatureCollection(/* draw water polygons */).map(function(f){return f.set('class',3)}); var bare = ee.FeatureCollection(/* draw bare polygons */).map(function(f){return f.set('class',4)}); var training_fc = urban.merge(agri).merge(forest).merge(water).merge(bare); var training = composite.select(bands).sampleRegions({collection:training_fc, properties:['class'], scale:10}); // Train Random Forest var classifier = ee.Classifier.smileRandomForest(100).train({ features: training, classProperty: 'class', inputProperties: bands }); // Classify var classified = composite.select(bands).classify(classifier); // Accuracy var validated = training.classify(classifier); var cm = validated.errorMatrix('class','classification'); print('Confusion Matrix:', cm); print('Overall Accuracy:', cm.accuracy()); print('Kappa:', cm.kappa()); // Visualize var palette = ['ef4444','f59e0b','10b981','06b6d4','a17040']; Map.centerObject(geometry, 11); Map.addLayer(classified, {min:0, max:4, palette:palette}, 'LULC'); Export.image.toDrive({ image: classified.byte(), description: 'LULC_Classification', folder: 'GEE_Exports', region: geometry, scale: 10 });
LST — Land Surface Temperature (Landsat-8/9)
// ═══ LAND SURFACE TEMPERATURE — Landsat-8/9 ═══ var geometry = ee.Geometry.Rectangle([83.2, 26.5, 83.9, 27.1]); var startDate = '2024-04-01', endDate = '2024-06-30'; // Load Landsat-9 (or use 'LANDSAT/LC08/C02/T1_L2' for Landsat-8) var landsat = ee.ImageCollection('LANDSAT/LC09/C02/T1_L2') .filterBounds(geometry).filterDate(startDate, endDate) .filter(ee.Filter.lt('CLOUD_COVER', 20)) .map(function(img) { var qa = img.select('QA_PIXEL'); var cloudMask = qa.bitwiseAnd(1<<3).eq(0).and(qa.bitwiseAnd(1<<4).eq(0)); return img.updateMask(cloudMask) .multiply(0.0000275).add(-0.2); // Scale factors }).median().clip(geometry); // Compute NDVI for emissivity var ndvi = landsat.normalizedDifference(['SR_B5','SR_B4']).rename('NDVI'); // Brightness Temperature from Band 10 var BT = landsat.select('ST_B10').multiply(0.00341802).add(149.0).subtract(273.15).rename('BT'); // Emissivity via NDVI threshold method var pv = ndvi.subtract(0.2).divide(0.5).pow(2).rename('Pv'); var em = pv.multiply(0.004).add(0.986).rename('emissivity'); // LST formula var lambda = 10.8e-6; // Band 10 wavelength var rho = 1.438e-2; var LST = BT.divide( ee.Image(1.0).add( BT.multiply(lambda/rho).multiply(em.log()) ) ).rename('LST'); // Visualize Map.centerObject(geometry, 11); var lstVis = {min:25, max:45, palette:['06b6d4','10b981','f59e0b','ef4444','7c2d12']}; Map.addLayer(LST, lstVis, 'LST (°C)'); Map.addLayer(ndvi, {min:0, max:0.8, palette:['red','yellow','green']}, 'NDVI', false); // Stats print('LST Stats:', LST.reduceRegion({reducer:ee.Reducer.mean().combine(ee.Reducer.minMax(),'',true), geometry:geometry, scale:30})); Export.image.toDrive({ image:LST, description:'LST_Celsius', folder:'GEE_Exports', region:geometry, scale:30 });
NDBI + Urban Heat Island — Sentinel-2
// ═══ NDBI + MNDWI + EVI — Sentinel-2 ═══ var geometry = ee.Geometry.Rectangle([83.2, 26.5, 83.9, 27.1]); var date1='2024-01-01', date2='2024-12-31'; var s2 = ee.ImageCollection('COPERNICUS/S2_SR_HARMONIZED') .filterBounds(geometry).filterDate(date1,date2) .filter(ee.Filter.lt('CLOUDY_PIXEL_PERCENTAGE',20)) .median().divide(10000).clip(geometry); // Compute all indices var NDBI = s2.normalizedDifference(['B11','B8']).rename('NDBI'); var NDVI = s2.normalizedDifference(['B8','B4']).rename('NDVI'); var MNDWI = s2.normalizedDifference(['B3','B11']).rename('MNDWI'); var EVI = s2.expression('2.5*(NIR-RED)/(NIR+6*RED-7.5*BLUE+1)', {NIR:s2.select('B8'),RED:s2.select('B4'),BLUE:s2.select('B2')}).rename('EVI'); var BSI = s2.expression('((SWIR+RED)-(NIR+BLUE))/((SWIR+RED)+(NIR+BLUE))', {SWIR:s2.select('B11'),RED:s2.select('B4'),NIR:s2.select('B8'),BLUE:s2.select('B2')}).rename('BSI'); // Urban mask var urban = NDBI.gt(0.1).and(MNDWI.lt(0.0)); // Stats var stats = ee.Image.cat([NDBI,NDVI,MNDWI,EVI,BSI]) .reduceRegion({reducer:ee.Reducer.mean(), geometry:geometry, scale:10, maxPixels:1e10}); print('Index Statistics:', stats); // Visualize Map.centerObject(geometry,11); Map.addLayer(NDBI, {min:-0.5,max:0.5,palette:['green','white','red']}, 'NDBI'); Map.addLayer(MNDWI, {min:-0.5,max:0.5,palette:['brown','white','cyan']},'MNDWI',false); Map.addLayer(urban, {min:0,max:1,palette:['white','orange']}, 'Urban Area', false); Export.image.toDrive({ image:ee.Image.cat([NDBI,NDVI,MNDWI,EVI]).float(), description:'SpectralIndices', folder:'GEE_Exports', region:geometry, scale:10 });
